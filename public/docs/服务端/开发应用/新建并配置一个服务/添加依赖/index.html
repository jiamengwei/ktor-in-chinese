    添加Ktor依赖 #

    本章节，我们将向你演示如何为已经存在的Gradle或Maven项目添加Ktor必须的依赖

    仓库配置 #

    添加Ktor依赖之前，你需要配置项目的依赖仓库：

    生产仓库 #

    Ktor的生产发布版在maven中心仓库可用，你可以像下面这样引用仓库

    1. Gradle(Kotlin)
    repositories {
    mavenCentral()
}
    1. Gradle(Groovy)
    repositories {
    mavenCentral()
}
    1. Maven
    Maven中央仓库已经包含相关依赖，无需额外添加

    预览版仓库（EAP） #

    你需要引用这个 仓库来访问 预览版本的Ktor：

    1. Gradle(Kotlin)
    repositories {
    maven {
        url = uri("https://maven.pkg.jetbrains.space/public/p/ktor/eap")
    }
}
    1. Gradle(Groovy)
    repositories {
    maven {
        url "https://maven.pkg.jetbrains.space/public/p/ktor/eap"
    }
}
    1. Maven
    <repositories>
    <repository>
        <id>ktor-eap</id>
        <url>https://maven.pkg.jetbrains.space/public/p/ktor/eap</url>
    </repository>
</repositories>

    注意：Ktor预览版可能需要 Kotlin开发版仓库：

    1. Gradle(Kotlin)
    repositories {
    maven {
        url = uri("https://maven.pkg.jetbrains.space/kotlin/p/kotlin/dev")
    }
}
    1. Gradle(Groovy)
    repositories {
    maven {
        url "https://maven.pkg.jetbrains.space/kotlin/p/kotlin/dev"
    }
}
    1. Maven
    <repositories>
    <repository>
        <id>ktor-eap</id>
        <url>https://maven.pkg.jetbrains.space/kotlin/p/kotlin/dev</url>
    </repository>
</repositories>
